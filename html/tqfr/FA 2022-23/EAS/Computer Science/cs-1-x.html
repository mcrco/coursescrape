Comments

Do you have any constructive comments for other students considering taking this course? In particular, comments about workload/distribution of the workload of the course, the necessity of the textbook, unexpected time requirements or flexibility, or unspecified prerequisites could be particularly helpful.

Projects pretty alright, we did some cool stuff. Workload is one project a week, varies in time requirements but usually only takes half a day. Much nicer and lower time requirements than cs1 – if you test in, take it.
Would recommend taking the class if given the option. Class is well-taught with engaging weekly projects that take the allotted time given on average.
Overall this was a great course. I learned Java in high school but kind of forgot it after I started using Python. This course helped me regain my knowledge in Java and will hopefully be a very good transition to CS 2.
At the beginning of the course, the assignments may seem a bit hard to understand/complete, but it gets better afterward if you sort out your time well. If you already know Java really well, the lectures are not as fruitful, but the code reviews really helped.
You will spend more than 5 hours on the later assignments so plan accordingly. However, do not hesitate to ask for extensions if you need it. Go to every office hours you can. The TAs are a gem.
If you passed the CS 1 placement test but don't feel entirely confident in your coding skills, I would highly recommend CS 1x. This course is taught in Java which is a nice refresher, especially if you haven't taken AP Computer Science or haven't done the APCS content in a while. Also, CS 2 is taught in Java so you will save yourself a bit of struggle at the beginning of CS 2 if you take CS 1x first.

CS 1x is a much smaller class (maybe 20 people) and so you will receive more direct instruction if that's what you're looking for. The class is very project-based and each weekly assignment is a project in of itself (ex, tictactoe, sudoku, stack calculator, etc). The class also emphasizes coding style, which I don't think CS 1 necessarily does. The TAs are extremely accessible and totally know their stuff. If you are stuck in the class, please don't hesitate to go to OH or ask them privately for help. Don't suffer all on your own (unless you actually want to).

The lectures are once a week and worth attending. I made the mistake of multitasking during lectures which drastically limited my learning. You have to really pay attention to class if you want to take things away from the lectures. Vanier actually teaches during lectures and really understands the concepts. He is also funny and passionate about the subject, which is a refresher compared to some of the core classes, where lectures may seem a bit boring to some people.

Note: the lectures are interesting but they are not actually directly applicable related to doing well in the assignments. The lectures are "nice to know" and tangentially related to the assignments, but in the end, your grade on the assignments is based on if you know how to code and if your code works, as opposed to if you understand certain definitions or memorize scenarios when the code will error.

Unless you're one of those CS geniuses that can finish an assignment in one sitting without any help, PLEASE start your sets earlier (preferably nearly done over the weekend), since the sets are due Tuesdays at 2am. It's hard to tell how long a set is until you put considerable effort into the set. I made that mistake and I regret all the moments I nearly had a heart attack.

TLDR; if you passed the placement test and don't want to suffer at the beginning of CS 2, take CS 1x. It's an informative, non-theory CS class that will give you a challenge.
The TAs are very helpful and asking questions both in and out of office hours is very important. The assignments themselves are relatively interesting as they go over a variety of concepts and development topics. Starting assignments early is also important to make use of office hours, but the TAs and professor are still willing to help outside of office hours. Overall a good course to take before CS2, but do not expect any review of Java syntax or concepts. Most topics covered are more like high level programming concepts than anything Java-specific.
The course is pretty interesting, although most of the syntax and documentation for new classes / methods was something you would have to search up on your own.
As Professor Vanier stated, "very chill class". You can usually do the assignments in 1 day or night.

Be sure to read the spec carefully to implement your code correctly. There are no test cases so you have to be the one to make sure your code works and is designed as intended.

The most important things I learned in this class were graphics and error handling. Other Java-specific things such as autonomous classes were nice to learn about but probably not too useful unless you decide to specialize in Java.

If you place into CS 001X, I would highly recommend taking as you will likely find the work minimal and the projects quite easy as compared to CS 001.
The format of this course was that every week there would be one lecture and one homework assignment, such as writing a program to play sudoku. The homeworks sometimes built on each other. The workload of this course probably varies a lot depending on your past programming experience. I took AP CS A with Java in high school and found the workload to be reasonable. Generally, the amount of hours I spent on the assignments varied wildly; some weeks the assignment only took a couple hours and other weeks it took a full day. They placed a lot of emphasis on style, so it was very difficult to obtain full points on an assignment even if your code worked perfectly. Conversely, you could still earn partial credit for code that didn't work. Overall I would definitely recommend taking this course over CS 1.
