The expected time per week in Lab is around 12-15hr. Prof. Niemeyer is very approachable, helpful and enthusiastic. His lectures are clear and fun.
The first half of the term is much more relaxed than the second half of the term. This was only the second year the class was offered but it was still pretty unorganized. Gunter, the professor, is really great though. He is very enthusiastic about it and genuinely wants students to learn. This was more of a CS class than anything else since all assignments were completed in Python. Over the course of the term you program a robot to fully autonomously explore and navigate a maze with line-following and ultrasonic sensors. The hardware used is very poor and gives bad data which makes the class very time consuming.

Not a single group was able to successfully complete all objectives laid out for the course, but the grading was extremely lenient. The first half of the term was chill and no more than a 9 unit course, but the last few weeks me and my partner consistently spent around 20 hrs each week. Don't take this class unless you are willing to spend a lot of time. It is a cool project that comes together at the end and rewarding to see work. You build up an entire code base over the term, starting with lower-level tasks like motor control and then work your way up to exploration algorithms.
The workload (at least for me) was somewhat high for this course, but your mileage may vary. I think I spent a lot of time making our code clean, but if you have more experience reading this kind of code and dealing with the logic you might have a much better time than I did. The better you think out your code before you start writing it and the cleaner it is in earlier weeks, the less time you will spend debugging later in this course. I would highly suggest planning out the code structure of the code you write with your teammates before starting to code. As far as I know, there is no textbook needed. Gunter also tends to record the lectures, so if you need to miss a lecture that is very helpful. Although, it always helped me to go to in-person lecture. This term the workload in some weeks was much more than in other weeks, but Gunter was flexible when the goals had to be scaled back during some weeks. I don't think there are any glaring prerequisites for this course, but a good coding background would help (also picking partners who produce clean/commented code). This year we only had lectures on Fridays, which was nice. The other parts of the week were structured as follows: The goals for the week would come out on Friday-Sunday (I imagine this will be earlier and more regular as Gunter flushes out the course some more) and we would pick a time on Thursday (at the beginning of term) to have our demos. That means for the most part I would work on Friday/Saturday to clean up code and fix demos from last week if we didn't meet the requirements, and then use Sunday-Wednesday to work on that week's goals. For most weeks we would write a report and tag our code on Gitlab to be graded by our designated TA. We would get a grade for the demo, the report, and the code, and then we were allowed to rework any of the three for some points back. I think having a designated TA was super helpful since you knew who they were and they would regularly read the code you have written. Overall, I thought it was a great course, and I had a lot of fun. It is amazing what you can do with a trio of IR and Ultrasound sensors on a tiny bot, and I look forward to taking more of Gunter's classes.
129 is a really well taught, well TA-ed class that covers some pretty cool concepts. Note that it is very programming intensive and bad code decisions at the start of the class can really come back to bite you towards the end of the class so make sure you can scope accordingly.
I found the workload for this class was somewhat high, in part due to how the class builds on itself. While every week had different goals, each built on top of the last, and so if issues with previous work cropped up, that would have to be fixed as well as adding new functionality.
As someone who had never programmed a robot before, I was a little nervous about taking this class, but I am so glad I took it! Don't be fooled by the ME/EE/CS cross listing, though, this is a CS class (python). Take this class with ample time to work on the weekly assignments. Gunter is a terrific prof and all the TAs are super passionate. I highly recommend to anyone with the slightest interest in robotics!
