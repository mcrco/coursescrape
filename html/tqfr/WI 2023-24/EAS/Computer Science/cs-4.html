What would you tell a student thinking about taking this course in the future?

Functional programming is fun and Vanier teaches the basics of it well. A lot of the material uses ideas from SICP, adapted to OCaml. I liked the special topic lectures near the end of the course, although a lot of it was a bit difficult for me to understand completely.

Vanier's choice in T-shirts improved the lectures tremendously. Overall, great and chill course.
This class is super straightforward and not too much work at all. Having reworks on every assignment makes this class less stressful and very possible to get an A/A+.

I didn't really like OCaml at all though. I feel like we could have learned more about functional programming instead of spending time on imperative and object oriented programming and trying to figure out the absolutely disgusting OCaml syntax to do things we know how to do in other languages.
Really nice class. OCaml is interesting and Vanier is great
I found Ocaml to be a super-odd language. It was weird learning the syntax at first, but then as the course went on, the code required less and less debugging. There were many times I wrote the code with no errors on the first try, which might just be a quality of ocaml. The last assignment and final are by far the most difficult, while the rest of the course sets can be done in one evening. Vanier is a good lecturer.
CS 4 is a very interesting class and OCaml is actually a fun language to learn. I am primarily a Python guy but learning OCaml was very intuitive and this class is very good at making sure you remember recursion and displaying how functional it is. Professor Vanier is a very good lecturer and I would recommend trying to go to most of the lectures as his example code is very interesting and helps with the problems sometimes.

What is really nice about this class is that if you pass all the tests and answer the questions correctly you will get an A+. It also very doable and easy to get this as each assignment except the midterm and final has reworks that you can get 100 on so you are basically guaranteed an A+ as long as you try on those and do well on the midterm/final (I also think the A+ cutoff was 95 so that is a really nice thing too). Overall, pretty chill class and interesting. You will be writing a lot of code and it takes time but it is nowhere near like CS 24 level of workload. I would put it at about CS 2 difficulty and workload probably....it's a fun and chill class to take your mind off other hard classes.
Great course, learned a lot of interesting topics about functional programming, and the homework assignments were all very reasonable. I took this course due to the CS major reqs, but was pleasantly surprised at how much I learned.

CS4 is a very interesting course that will teach you a new way of programming. Some programmers will struggle with this, and I know many who took to it almost immediately. By the end, you will definitely get the hang of it. Prof.Vanier's enthusiasm is refreshing, and he is genuinely one of the most understanding and kind professors at Caltech. Honestly, you could take the course just to learn from him and experience that kind of leniency with due dates, especially when term gets hard. I would definitely recommend starting the labs early, and doing a little bit everyday. Piling it on one night is rough, especially for some of the longer labs. Vanier is extremely understanding with extensions, but don't let the reworks pile up too much.
Taking this with CS21 was rough this year especially since this class's OH coincided with CS21 OH a lot. But it's manageable. Even with getting less than 50% on a whole lab, you could probably swing an A-. If you manage your time and work well, this class should be one of the easier A's. Prof Vanier takes pride in that, and rightfully so.
