Comments

What would you tell a student thinking about taking this course in the future?

Not a good course. Not well organized, didn't learn much
The course provides an overview of classical control theory. However, the topics are often taught quickly and students don't have enough time to process everything.
The HWs are important to systematize the concepts, but be ready to spend several hours debugging Python code and/or understanding how to use tools of the controls library. I think that there was too much emphasis on the use of Python to solve controls problems, which was not clearly explained/detailed at the beginning.
I would not recommend this class at all. The professor breezes through the information far too quickly and doesn't cover anything in any semblance of detail. There is a significant Python component of the class, and the code is buggy, poorly explained, and difficult to use and implement. You would be better off taking a controls class that focuses more on the theory of how control systems work, since this class is basically a coding class and you will learn nothing about the math behind common control systems.
I would highly recommend taking this course with Richard as the instructor. His deep knowledge of the subject matter, passion for teaching, and dedication to providing excellent course materials create an engaging and supportive learning environment. The course is well-structured, with a good balance of theory, computational exercises, and practice problems to prepare you for the final assessment. While the control package may take some time to get accustomed to initially, Richard's clear explanations and willingness to help will guide you through it. The workload is manageable, and the problem sets are well-designed to reinforce your understanding. Overall, this course offers a valuable opportunity to learn from an expert in the field and gain practical skills in feedback control systems.
This class is alright, but it is extremely time consuming and I had to panic take it on pass fail. The final exam format was not expained till last minute, and the final exam prep was an afterthought. Did not feel like an actual introductory course. Sets consistently took me 14+ hours only to get an 80% and not really any helpful feedback. TA's did not know how to help most of the time, so office hours were spent amongst students helping eachother.
Definitely not as easy as it has been in years past, but I think that's for the better. A better professor, better content, and overall a more interesting and helpful class. This class helped me realize both how important controls are and also how much I enjoy controls. This class is by no means easy, but it absolutely is worth your time.
This course walks you through a lot of the key concepts in modern day control systems. It is challenging but if you stick it through you'll have a good grasp on the important concepts and tools for >90% of control systems.
Be comfortable with python. The class is basically entirely based on it. Also, be ready to just sit for hours plugging stuff into the black box until something decent gets spit out.
Class was extremely coding intensive, so would recommend having a strong Python background (& ability to navigate documentation). I felt like there was less emphasis on the conceptual aspect and more on the implementation.
