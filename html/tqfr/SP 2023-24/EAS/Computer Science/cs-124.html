What would you tell a student thinking about taking this course in the future?

This class was very painful
Be prepared to do nothing but code in c for hours and hours every week. Personally, found out way too late that I did not have the time I thought I needed to take this class. I ended up spending like 40 hours a week once for VM. There are 5 projects, lectures 3 times a week, and OH a couple times a week. The TAs were sometimes helpful but it could be a bit difficult since there are so many ways to go about implementing the projects. Prof. Pinkston is funny and engaging, which is great for learning the concepts. Unfortunately, this class sucked the life out of me otherwise. I'd say what I learned was still pretty useful for understanding backend / operating systems in a practical sense. I would not suggest taking this if you're taking 4+ classes and looking for work-life balance. You won't find it. Definitely don't take it if you didn't enjoy CS 24.
Highly highly recommend you take this class! It is very high workload though so be prepared, and take it with a group you have worked with before.
This was a great class. It was definitely one of the hardest courses I've taken at Caltech, but potentially my favorite, and the one of the most worth. You'll learn a ton about how computers work, and everything you learned in CS24 will make much more sense. Highly recommend taking this course with an otherwise low-effort schedule though since this course will be time-consuming, regardless of your expertise. Expect to spend about 15-20 hours a week on the projects, perhaps more if you run into difficult bugs. Thus, make sure you find a good teammate (or two) that are committed to the class and willing to make the same time commitment.

The projects vary significantly in difficulty. The Command Shell project was essentially a warmup. The Threads project and User Programs project were much more difficult. And then the Virtual Memory project (probably the worst, many teams did not finish) and File Systems project were extremely difficult. The last two projects are much more conceptually difficult, harder to debug, and required more code (1000+ lines). Keep this in mind when using your late tokens (you have 6 one-day late tokens).

Office hours are useful for this class, but mostly just for conceptual questions, which was understandable. The projects are very open-ended, they're cumulative, and every team has a different implementation, so they won't be able to debug your code very easily.

Overall though, I'd recommend the course and if you're thinking of taking it, I'd say don't be intimidated to take the class. I understood literally nothing from CS24 and still found the projects in this class to be approachable because this class is much better taught and structured.

This class is great, but please heed the TQFRs—it's a massive time commitment! Do not take this course on an otherwise heavy term, or you will have a terrible time. Optimally, this will be your only "real" or underunited class, because the projects take a long time to understand, write, and debug (20+ hours/person/project in our case).

You learn so much about how computers work from the lectures and the projects. You don't need to prep beforehand—I hadn't used C in over a year and picked it up quickly with the first project, which is intentionally easier than later projects. Note that the final 2 projects (VM/filesys) are certainly the hardest, so I would start those two even earlier than you start the first few.
